## 核心设计亮点

### 1. 多武器系统
**最多同时携带6种武器，所有武器同时自动攻击**（特殊角色如多面手可携带12把）
- 区别于传统射击游戏
- 创造独特的构筑深度
- 提供强烈的视觉和心理满足感
- **武器总数**：近战33种 + 远程22种 = 55种

### 2. 精妙的数值平衡
- 怪物与玩家同步成长（基础数值+成长数值）
- 属性的边际递减效应
- 强力效果伴随代价（正负属性平衡）
- 没有绝对最强的选择

### 3. 掉落保留机制
**未拾取的金币和材料下一波重新掉落（标记"x2"）**
- 降低失误惩罚
- 保持风险-收益决策
- 培养资源经营意识

### 4. 商店刷新递增成本
**可以无限刷新，但成本递增，每波重置**
- 允许寻找理想商品
- 防止无限刷新
- 创造决策压力

### 5. 目标优先级的自然形成
**通过敌人机制让玩家自然产生击杀优先级**
- 强化型敌人（不杀会让其他敌人变强）
- 高速敌人（快速接近，威胁高）
- 不需要UI标记，玩家自然学习

### 6. 收获属性的阶段性价值
**前期投资收获→中期平衡→后期放弃收获专注战斗**
- 创造资源规划的策略深度
- 鼓励玩家策略转型
- 避免单一最优策略

### 7. 快节奏体验
- 20波固定结构
- 每局20-30分钟
- 战斗+商店交替，张弛有度
- 适合碎片化时间

### 8. 高重玩价值
- 30+角色，每个玩法不同
- 数十种武器和道具
- 随机性保证每局体验不同
- 无数种构筑组合

## 与我们项目的对比要点

### 题材对比
- **Brotato**：土豆对抗外星人（幽默、轻松）
- **我们的项目**：星河舰长（太空、科幻、严肃）

### 视角对比
- **Brotato**：俯视角
- **我们的项目**：待定（可能需要明确）

### 核心机制对比
- **Brotato**：多武器同时使用、自动射击
- **我们的项目**：舰船模块系统、舰载武器系统（需要明确独特性）

### 可以借鉴的地方
1. **20波结构**：控制游戏时长
2. **战斗-商店循环**：张弛有度的节奏
3. **掉落保留机制**：友好的容错设计
4. **递增刷新成本**：防止滥用
5. **目标优先级设计**：通过敌人机制自然形成
6. **收获属性**：阶段性价值的经济设计
7. **正负属性**：强力效果伴随代价

### 可以差异化的地方
1. **更强的叙事**：Brotato叙事较弱，我们可以加强世界观和剧情
2. **更复杂的关卡**：Brotato关卡相对单一，我们可以设计更多样的关卡
3. **舰船成长**：长期进度系统，Brotato主要是单局游戏
4. **模块化系统**：舰船模块可能比单纯武器更有深度
5. **多元化敌人**：不同势力、不同机制
6. **战役模式**：而非单纯的20波生存
